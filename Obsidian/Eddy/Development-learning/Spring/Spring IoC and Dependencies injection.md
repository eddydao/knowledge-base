# Dependency inversion
Là một nguyên lý để thiết kế và viết code 
Đảm bảo hai yếu tố sau:
	- Các module cấp cao không nên phụ thuộc vào các module cấp thấp mà nên phụ thuộc vào abstraction
	- Các class giao tiếp thông qua interface, không thông qua implementation

# Inversion of Control
Một design pattern tuân thủ theo Dependency inversion
Có nhiều cách thực hiện design pattern này:
- Service locator
- Event
- Delegate
- Dependency injection
